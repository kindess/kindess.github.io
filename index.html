SpringBoot与Angular前后端分离之携带Cookie

跨域

SpringBoot支持的跨域方式

支持四种方式：
在Controller层使用注解（@CrossOrigin）
实现WebMvcConfigurer接口，全局跨域（或者继承其实现类）
使用Filter方式，进行全局跨域
手工设置响应头（HttpServletResponse）

关于Cookie跨域需要了解

关于Cookie的path与Domain
Cookie中的httponly的属性和作用

除此之外，还需要注意的是：
后端必须需要设置Access-Control-Allow-Credentials，因为安全原因，跨域时，后端响应头不会自动携带Cookie，
只有设置此属性才会携带Cookie。而前端必须设置withCredentials，前端才会去拿后端携带的Cookie。
我直接截一张别人博客上的图来说明！简单，明了！看图

关于Springboot使用了Security的RememberMe功能进行跨域导致的问题

上面所说的后端设置Access-Control-Allow-Credentials，前端设置withCredentials就能解决问题啦！但是，光是上面
的设置是无论如何也解决不了前端无法获取Security的RememberMe功能生成的Cookie！

回到上面所讲的，Cookie中httponly的属性决定了前端是否能通过JS去获取并操作Cookie，而如果Springboot使用了
Security的RememberMe，那么会遇到一个问题，那就是这个RememberMe功能的Cookie，默认设置了httponly为true。

通过AbstractRememberMeServices源码分析，如下图


属性setHttpOnlyMethod决定了Cookie的httpOnly，如下图


通过DeBug进行源码执行时，在对判断属性setHttpOnlyMethod进行判断之前（查看断点处），cookie的httpOnly还是false，
点击执行下一步时，”this.setHttpOnlyMethod != null”表达式结果将为真，会执行if中的代码。然后继续下一步，就执行到了
最后的”response.addCookie(cookie);”，此时，cookie的httpOnly就变成了true了！
前端将获取不到这个cookie，如图

啊？是我跨域的时候没有传Cookie吗？不，不是的！

查看Chrome的Appliction面板，发现浏览器中一个”rememberMe”的cookie，这就是我利用Security的RememberMe功能生成的Cookie!
那浏览器中有，为啥”document.cookie”还拿不到呢？注意了！Application面板中，我箭头标记的”http”就是httpOnly!那个”√”就是表示httpOnly=true!代表你永远也拿不到的痛！

既然知道问题了，还是要解决的。将cookie的httpOnly设置为false，这是不可行的，因为AbstractRememberMeServices没有提供一个我们设置setHttpOnlyMethod
属性的方式（因为它是一个私有属性），如果非要修改这个值，只能重写AbstractRememberMeServices整个类。。。或者采用反射机制，修改父类属性值！

Angular入门学习一

安装环境

安装node.js

学习Angular，首先需要安装node.js，需要node内置的npm支持，node.js的官网下载

安装angular

使用node安装angular：
npm的官网在国外，同docker等等，网速太慢会引起各种问题，因此使用node安装之前，先换源，使用淘宝镜像安装。

npm config set registry https://registry.npm.taobao.org //淘宝换源指令 npm install -g @angular/cli //安装angular

查看版本信息，是否安装成功，运行命令 ng v

新建一个angular空项目

ng new 项目名

使用ng命令新建一个angular空项目。不过，使用上面这条命令创建会出现问题（没遇到就算了，遇到请使用下面的方式创建）

ng new --skip-install

如图，

上面我们没有安装angular的依赖，因此我们进入项目根目录下，我这个项目根目录是C:\Users\Administrator\my-app,
打开cmd命令，执行npm install或者cnpm install（上面换源了才能使用cnpm命令）
运行命令

ng serve

打开浏览器，访问localhost:4200，如果执行命令出现类似下面的报错，

An unhandled exception occurred: Cannot find module '@angular-devkit/build-webpack' See "C:\Users\ADMINI~1\AppData\Local\Temp\ng-lEA4YS\angular-errors.log" for further details. npm ERR! file C:\Windows\system32\cmd.exe npm ERR! code ELIFECYCLE npm ERR! errno ENOENT npm ERR! syscall spawn npm ERR! test@0.0.0 start: `ng serve` npm ERR! spawn ENOENT npm ERR! npm ERR! Failed at the test@0.0.0 start script. npm ERR! This is probably not a problem with npm. There is likely additional logging output above.

npm ERR! A complete log of this run can be found in:
npm ERR! D:\Program Files\nodejs\node_cache_logs\2019-07-11T06_13_09_014Z-debug.log

表示缺少依赖，最直接的办法就是删掉my-app项目中的文件夹node_modules，这文件夹下全是执行cnpm install
命令安装的依赖，删掉之后我们重新执行cnpm install进行重新下载。如仍有报错，继续。。。
